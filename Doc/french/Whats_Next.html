Futur

BeBuilder V0.5 est toujours considéré en pré-release. Il n'est donc pas terminé! Nous sommes toujours intéressés par tous les commentaires même les mauvais. :-)
  • Tous les BControl contenus dans le BeBook

    Comme nous le disons depuis la version V0.1 Nous travaillons fort à l'implantation de tous les composants BeOS dans BeBuilder. C'est juste une question de temps! 8D

  • Options dans le Projet

    Depuis V0.5 on peut charger et sauvegarder des projets.

  • Integration avec GCC

    Le menu Tools a été modifié mais est toujours experimental vous pouvez compiler, executer et editer vos fichiers sources depuis BeBuilder. Désolé pour les fanatiques de Pe ou de Eddie, seul le BeIDE est supporté pour le moment.

  • Génération dynamique de code

    Comme vous l'avez sûrement remarqué, le menu Generate n'existe plus. Depuis la version V0.5 la génération de code est dynamique, aussitôt que vous déposez un composant ou que vous modifiez une propriété le code source est modifié et vos modifications devraient être conservées.
    Je préfère dire devraient pour ne pas prendre de risque mais une grosse partie de BeBuilder est développé avec BeBuilder...
    UNE RÈGLE : NE JAMAIS MODIFIER UNE LIGNE DE CODE GÉNÉRÉ PAR BEBUILDER.

  • Bugs

    Vous devriez trouver des bugs... :) N'hésitez pas à m'envoyer une desciption détaillée de ces bugs avec si possible un projet qui pose problèmes.

  • Editeur

    Nous espérons ajouter un éditeur très simple dont le code sera fourni pour supporter les améliorations de BeBuilder. Avis aux amateurs.

  • Ouvertures

    Pour le moment BeBuilder ne génère que du C++, amateurs de Delphi nous aimerions avoir une version Pascal et pourquoi pas Java de BeBuilder!
